Quickstart

Run AI coding agents safely.

Modonome runs beside your repository. It starts disabled and in dry-run, so every step below is safe. You preview what it would do, adopt local state when ready, and prove the controls before autonomy is ever armed.

✓ Off by default · nothing is written until you choose

STEP 1
See what it would do

The dry run reads your repository, detects the stack and gates, lists protected paths, and proposes a few bounded pieces of work. It writes nothing.

# read-only, requires the published npm package
npx modonome dry-run .

STEP 2
Add local state

Scaffolding creates a .modonome/ directory with a config and state files, all disabled and in dry-run. It leaves your existing files untouched. Add --write to apply, or run without it to preview. If your repo already uses .autonomy/, Modonome adopts it instead.

npx modonome scaffold . --write
→ .modonome/ created  ·  autonomy_enabled: false

STEP 3
Review and validate

Open .modonome/config.yaml. Confirm the protected paths, the gates, and the caps. Keep autonomy_enabled and auto_merge off. Then validate.

npx modonome validate .modonome/config.yaml

STEP 4
Prove the controls

Run AgentProof to confirm every safeguard holds on your own setup. It runs a real adversarial attempt for each scenario and confirms it is caught.

node agentproof/runner.mjs
→ Score: 25/25 normative | 10/10 extended (35/35 total)  Level: HARDENED

Read what each scenario proves on the AgentProof page.

Arming later

Armed mode is a deliberate, owner-only step. Autonomy turns on only when an owner sets MODONOME_ARMED=true in the CI or harness environment, and only after branch protection, required CI checks, and code owners are in place. Until then, Modonome stays at rest and proposes small, fenced, independently checked changes for human review.

Every change stays small and pinned by a test, the maker and checker are always different roles, and the gates run in your CI where the agent cannot edit them. That is what keeps autonomy trustworthy.

The full guide, including the complete arming checklist, is in the QUICKSTART documentation. Back to the Modonome home page.
